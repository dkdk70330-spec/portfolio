# AI 캐릭터 포트폴리오 1단계 프로토타입

백엔드 없이 실행되는 개인용 정적 포트폴리오입니다.

## 실행

가장 간단한 방법은 `index.html`을 더블클릭하는 것입니다.
브라우저 보안 설정에 따라 로컬 파일의 일부 동작이 제한되면 아래 명령으로 로컬 서버를 실행하세요.

```bash
python -m http.server 8000
```

그 후 브라우저에서 `http://localhost:8000`을 엽니다.

## 데이터 수정

`assets/data.js`의 `window.PORTFOLIO_DATA.characters` 배열을 수정합니다.

캐릭터마다 다음 항목을 가집니다.

- 이름과 한 줄 소개
- 상세 설명
- 장르와 태그
- 이미지 여러 장
- 플랫폼별 외부 링크
- 추가 콘텐츠 여러 개와 콘텐츠별 스포일러 설정
- 대표 캐릭터 여부

## 이미지 교체

`images` 폴더에 이미지를 넣고 `assets/data.js`의 파일명을 변경합니다.
현재 이미지는 프로토타입용 SVG 플레이스홀더입니다.

## 포함 기능

- 제작자 프로필과 외부 링크
- 대표 캐릭터 영역
- 캐릭터 10명
- 이름·소개·태그 검색
- 장르 필터
- 플랫폼 필터
- 반응형 캐릭터 카드
- 상세 모달
- 이미지 갤러리
- 플랫폼 이동 링크
- 제작 비하인드·과거·후일담·비밀 설정 콘텐츠
- 콘텐츠별 스포일러 경고와 접기/펼치기
- 밝은/어두운 테마

## 플랫폼 아이콘 관리

플랫폼은 `assets/data.js` 상단의 `platforms` 목록에서 관리합니다.

```js
platforms: [
  { id: "lune", name: "Lune", icon: "platforms/lune.svg" }
]
```

- `id`: 캐릭터 데이터에서 사용하는 고정 식별자
- `name`: 마우스를 올렸을 때 표시되는 플랫폼 이름
- `icon`: `images` 폴더를 기준으로 한 아이콘 경로

캐릭터에는 플랫폼 ID와 해당 캐릭터의 실제 주소만 입력합니다.

```js
platforms: [
  { id: "lune", url: "https://example.com/lune/character" }
]
```

아이콘을 바꾸려면 `images/platforms` 안의 SVG 파일을 같은 이름으로 교체하거나, `icon` 경로를 새 파일명으로 변경하세요.


## 캐릭터 추가 콘텐츠

캐릭터의 `contents` 배열에 제작 비하인드, 과거, 못다한 이야기, 비밀 설정 등을 원하는 만큼 추가합니다.
배열에 입력한 순서대로 상세 화면에 표시됩니다.

```js
contents: [
  {
    id: "behind-scenes",
    type: "제작 비하인드",
    title: "이 캐릭터를 만들게 된 계기",
    content: [
      "첫 번째 문단입니다.",
      "두 번째 문단입니다."
    ],
    spoiler: false
  },
  {
    id: "hidden-past",
    type: "과거",
    title: "숨겨진 과거",
    content: ["스포일러 본문입니다."],
    spoiler: true,
    warning: "중반부 사건에 대한 스포일러가 포함되어 있습니다."
  }
]
```

- `type`: 제작 비하인드, 과거, 못다한 이야기, 비밀 설정 등의 분류
- `title`: 상세 화면에 표시되는 콘텐츠 제목
- `content`: 문단 배열
- `spoiler`: `true`이면 경고 카드로 접어서 표시
- `warning`: 스포일러 카드의 안내 문구

이전 버전의 단일 `spoiler` 객체도 화면 코드에서 자동 변환해 표시하지만, 새 데이터는 `contents` 배열 사용을 권장합니다.
